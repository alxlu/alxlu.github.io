#Hyperion.ai

Hyperion.ai is the product of a startup I am currently the co-founder and CEO of. It is a custom multivitamin that improves itself over time by using artificial intelligence to continuously monitor and analyze a user's health data. This writeup will be focused primarily on the business, technology, and other behind the scenes aspects of Hyperion.ai. For more information about the product itself, check out: http://hyperion.ai

#Overview

People come in all different shapes and sizes. So, it’s no surprise that we all have unique nutritional needs. The nutritional supplement industry has capitalized on this problem, but we feel that current solutions leave a lot to be desired. Today, when consumers shop for vitamins-whether from a local drugstore or online-they are bombarded with tons of options. How are they supposed to know which product meets their needs? And perhaps more importantly, how are they supposed to know if said product provides a tangible improvement to their quality of life? That’s where we come in.

Hyperion AI provides a completely free app that collects a user’s health data. The app allows users to track any metric they’d like. Additionally, users can connect fitness integrations such as HealthKit and Fitbit. We then visualize these metrics so that the user can see their progress over time. On top of that, we also normalize the data, allowing users to compare different metric against one another. For example, a user might wonder if their overall energy levels correlate with the amount of water they drink. With Hyperion AI, it’s only a few taps away. Finally, Hyperion AI will use artificial intelligence to continuously process the user’s health data, allowing us to find potentially significant correlations between tracked metrics. If the user is subscribed to one of our monthly supplement plans, the AI will also construct and continuously refine a unique, custom supplement tailored for each user, which will ship on a monthly basis.

We believe that by eliminating the financial barrier to entry, our free app gives us a huge edge against our competitors, building a relationship of trust with our users before they give us a dime.

#Tech Stack

#App & Web UI

Hyperion.ai is a React Native App. And the Web UI is written using React. For both platforms we use Redux to manage state and Redux-Saga for side effects (e.g. API calls). This allows us to share a lot of the business logic despite the differences between React Native and React (web) components. Additionally, we also use the iOS WebView element to display several views in our app. This allows us to develop the mobile site and iOS app simultaneously while giving us the option to implement the view natively. So far, despite HTML5-based apps reputation for being less performant than native, the difference in performance for our app is imperceptible. I believe this is due to a combination of factors:

  • Improved browser rendering engines
  • The nature of our app (e.g. we don't need to perform any expensive animations)
  • Improved hardware

Currently, I only use React Native views primarily to provide a more native-like experience. For instance, the ability to swipe to go back to a previous page is something easily done using React Native, but is difficult (and likely less performant using HTML5).

#Service

The Hyperion.ai backend is written using Node.js and the database is hosted using PostgreSQL. For the backend, I decided to leverage several services to help speed up development and potentially reduce headaches down the line—particularly regarding security. We are using Heroku to host the service.

#Tooling & Deployment

I set up Jenkins to handle running out automated tests and deployment. Whenever a change is made to the Deployment branch in either of the projects, Jenkins will run all our automated tests and if they pass, it will deploy them to production (AWS for the frontend, Heroku for the backend).
